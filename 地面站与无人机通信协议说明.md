# 地面站与无人机通信协议说明

本系统由地面站和无人机构成，地面站负责发送任务控制指令、接收无人机状态与盘点结果，并在界面上实时显示任务进度和货物盘点信息。通信内容围绕 D 题中“遍历盘点”和“定向盘点”两类任务展开，满足地面站实时显示货物编号与坐标信息的要求。

## 一、通信方向

系统通信分为两个方向：

1. **地面站 → 无人机**
    地面站向无人机发送任务启动、目标扫描和紧急停止等控制指令。
2. **无人机 → 地面站**
    无人机向地面站回传节点状态、任务状态、二维码识别结果、目标货物信息以及任务回复信息。

------

## 二、地面站发送指令

目前地面站共向无人机发送 **4 条指令**。

| 指令内容                | 含义           | 用途                                  |
| ----------------------- | -------------- | ------------------------------------- |
| `CMD:START_TASK1`       | 启动任务 1     | 开始遍历盘点任务                      |
| `CMD:START_TASK2`       | 启动任务 2     | 开始定向盘点任务                      |
| `CMD:TASK2_SCAN_TARGET` | 扫描目标二维码 | 在任务 2 起飞前识别抽取的目标货物编号 |
| `CMD:EMERGENCY_STOP`    | 紧急停止       | 触发无人机紧急刹停或降落流程          |

其中，任务 1 对应对 A、B、C、D 四个货架面共 24 个固定坐标位置的完整巡查；任务 2 对应根据目标编号执行定向盘点。

------

## 三、地面站接收消息

目前地面站接收无人机发送的消息共分为 **5 类**。

------

## 1. 状态消息：`STATUS:xxx`

状态消息用于表示无人机或机载节点当前所处的工作状态。地面站接收到该类消息后，会更新界面状态灯、日志信息或任务进度显示。

### 已处理状态值

| 状态消息                  | 含义                   |
| ------------------------- | ---------------------- |
| `STATUS:VISION_READY`     | 视觉节点已就绪         |
| `STATUS:RECEIVER_READY`   | 通信接收节点已就绪     |
| `STATUS:FSM_READY`        | 飞控状态机已就绪       |
| `STATUS:BOOT_WAITING`     | 无人机已上电，等待启动 |
| `STATUS:TAKEOFF`          | 无人机正在起飞         |
| `STATUS:TAKEOFF_OK`       | 起飞完成               |
| `STATUS:TASK1_RUNNING`    | 任务 1 正在执行        |
| `STATUS:TASK2_RUNNING`    | 任务 2 正在执行        |
| `STATUS:MISSION_RUNNING`  | 航线任务正在执行       |
| `STATUS:SCANNING`         | 正在扫描二维码         |
| `STATUS:LANDING`          | 正在降落               |
| `STATUS:MISSION_FINISHED` | 航线任务完成           |
| `STATUS:TASK_FINISHED`    | 当前任务完成           |
| `STATUS:ERROR`            | 系统出现错误           |

该类消息主要用于让地面站实时掌握无人机状态，避免用户在任务未准备好、未起飞或异常状态下误操作。

------

## 2. 盘点结果消息：`SCAN:<坐标>:<编号>`

盘点结果消息用于回传无人机识别到的货物编号及其对应的固定坐标。

### 标准格式

```
SCAN:A1:17
```

表示无人机在固定坐标 `A1` 处识别到货物编号 `17`。

### 兼容简化格式

```
A1:17
```

地面站也兼容这种简化格式，含义同样是：

```
坐标 A1 对应编号 17
```

该消息主要用于任务 1 的遍历盘点。地面站收到后，会更新盘点结果表，并记录坐标与编号之间的映射关系。

------

## 3. 目标编号消息：`TARGET_ID:<编号>`

目标编号消息用于任务 2 起飞前的目标识别阶段。

### 示例

```
TARGET_ID:17
```

表示无人机已经识别到抽取二维码中的目标货物编号为 `17`。

地面站收到该消息后，会在界面上显示目标编号，并为后续定向盘点任务提供目标依据。

------

## 4. 目标盘点结果消息：`TARGET_RESULT:<坐标>:<编号>:<结果>`

目标盘点结果消息用于任务 2 完成目标货物识别后，向地面站回传最终结果。

### 示例

```
TARGET_RESULT:A1:17:OK
```

表示无人机在坐标 `A1` 找到编号为 `17` 的目标货物，结果为 `OK`。

该类消息通常包含三个部分：

| 字段     | 含义                 |
| -------- | -------------------- |
| `<坐标>` | 目标货物所在固定坐标 |
| `<编号>` | 目标货物编号         |
| `<结果>` | 识别或盘点结果       |

地面站收到后，会显示目标货物的坐标、编号和最终盘点状态。

------

## 5. 回复消息：`REPLY:xxx`

回复消息用于无人机对地面站指令或航线操作进行确认。地面站接收到该类消息后，会在日志区显示对应提示。

### 有专门提示的回复

| 回复消息               | 含义          |
| ---------------------- | ------------- |
| `REPLY:TASK1_STARTED`  | 任务 1 已启动 |
| `REPLY:TASK2_STARTED`  | 任务 2 已启动 |
| `REPLY:MISSION_SAVED`  | 航线已保存    |
| `REPLY:MISSION_LOADED` | 航线已加载    |
| `REPLY:TAKEOFF_OK`     | 起飞确认完成  |
| `REPLY:SCAN_OK`        | 扫描确认完成  |

对于其它 `REPLY:xxx` 格式的消息，地面站统一显示为“收到回复”，用于提示用户无人机端已经返回响应。

------

## 四、通信流程概述

### 任务 1：遍历盘点流程

1. 无人机上电后发送状态消息，例如：

```
STATUS:BOOT_WAITING
```

1. 各节点启动完成后，依次发送：

```
STATUS:VISION_READY
STATUS:RECEIVER_READY
STATUS:FSM_READY
```

1. 用户在地面站点击任务 1 启动按钮，地面站发送：

```
CMD:START_TASK1
```

1. 无人机回复任务启动：

```
REPLY:TASK1_STARTED
```

1. 无人机开始起飞和巡查，回传状态：

```
STATUS:TAKEOFF
STATUS:TAKEOFF_OK
STATUS:TASK1_RUNNING
STATUS:MISSION_RUNNING
```

1. 每识别到一个二维码，回传盘点结果：

```
SCAN:A1:17
SCAN:A2:03
SCAN:B1:12
```

1. 任务完成后，回传：

```
STATUS:MISSION_FINISHED
STATUS:TASK_FINISHED
STATUS:LANDING
```

------

### 任务 2：定向盘点流程

1. 任务 2 起飞前，用户点击目标扫描按钮，地面站发送：

```
CMD:TASK2_SCAN_TARGET
```

1. 无人机识别抽取二维码，并回传目标编号：

```
TARGET_ID:17
```

1. 用户启动任务 2，地面站发送：

```
CMD:START_TASK2
```

1. 无人机回复：

```
REPLY:TASK2_STARTED
```

1. 无人机执行定向盘点，过程中回传：

```
STATUS:TASK2_RUNNING
STATUS:SCANNING
```

1. 找到目标货物后，回传目标盘点结果：

```
TARGET_RESULT:A1:17:OK
```

1. 任务结束后，回传：

```
STATUS:TASK_FINISHED
STATUS:LANDING
```

------

## 五、异常与安全处理

地面站设置紧急刹停按钮。当出现飞行异常、识别异常或现场需要立即终止任务时，地面站可向无人机发送：

```
CMD:EMERGENCY_STOP
```

无人机接收到该指令后，应立即停止当前任务，并进入安全处理流程，例如悬停、降落或执行预设的紧急降落动作。

同时，当无人机端检测到异常时，可主动发送：

```
STATUS:ERROR
```

地面站收到后应在日志区显示错误信息，提醒操作人员及时处理。

------

## 六、通信协议总结

本系统通信协议采用字符串消息格式，结构简单、易于调试，能够满足地面站与无人机之间的任务控制、状态同步和盘点结果回传需求。

地面站当前发送 **4 类控制指令**，包括任务 1 启动、任务 2 启动、目标扫描和紧急停止；接收 **5 类无人机消息**，包括状态消息、盘点结果消息、目标编号消息、目标盘点结果消息和回复消息。

通过该通信设计，地面站可以实时显示无人机工作状态、货物编号、货物坐标以及任务执行结果，为立体货架自主盘点任务提供了稳定的信息交互基础。
